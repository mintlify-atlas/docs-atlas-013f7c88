## What is Kairo?

Kairo is an intelligent learning ecosystem built for the Riwi bootcamp program. It analyzes each student's academic performance and soft skills to generate dynamic, personalized 4-week growth paths — ensuring no coder gets left behind.

The platform serves two types of users:

- **Coders** — Bootcamp students who receive personalized AI-generated study plans, interactive exercises, and feedback from their Team Leader
- **Team Leaders (TLs)** — Mentors who use analytics dashboards and AI-generated reports to proactively identify and support at-risk students

## Tech stack

| Layer | Technology |
|-------|-----------|
| API Gateway | Node.js + Express 5 |
| AI Microservice | Python 3 + FastAPI |
| LLM | Groq (llama-3.3-70b-versatile / Qwen2.5-7B-Instruct) |
| AI Orchestration | LangChain |
| Database | PostgreSQL 14+ via Supabase |
| Frontend | Vanilla JS + Bootstrap 5 |
| Auth | Session-based + Passport (Google, GitHub OAuth) |

## Architecture overview

Kairo uses a **hybrid two-backend** architecture:

1. **Node.js (Port 3000)** acts as the system orchestrator — managing user sessions, authentication, business logic, and proxying AI requests to the Python service.
2. **Python/FastAPI (Port 8000)** acts as the AI brain — processing learning data through the LLM and returning structured JSON plans, exercises, and reports.
3. **Frontend (Port 5500)** is a static multi-page application (MPA) with separate views for coders and Team Leaders.

<Info>
  Both backend services must be running simultaneously for Kairo to work. The Node.js service communicates with the Python service internally via `PYTHON_API_URL`.
</Info>
