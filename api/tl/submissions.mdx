---
title: 'Evidence submissions'
description: 'Review and score evidence submissions from coders.'
---

Coders submit their exercise solutions for TL review. TLs can provide feedback on submissions. Reviewing a submission awards the coder **+15 Kairo points** via the `tl_approved` scoring event.

---

## List submissions

```
GET /api/tl/submissions
```

Returns the most recent 50 exercise submissions from coders in the TL's clan, ordered by submission date descending.

<Info>
  Results are automatically scoped to the TL's clan. A TL cannot view submissions from coders in other clans.
</Info>

### Response — 200 OK

<ResponseField name="submissions" type="object[]">
  <Expandable title="item properties">
    <ResponseField name="id" type="integer">Submission ID.</ResponseField>
    <ResponseField name="code_submitted" type="string">The code the coder submitted.</ResponseField>
    <ResponseField name="submitted_at" type="string">ISO 8601 submission timestamp.</ResponseField>
    <ResponseField name="tl_feedback_text" type="string | null">TL's review feedback, or `null` if not yet reviewed.</ResponseField>
    <ResponseField name="reviewed_at" type="string | null">ISO 8601 timestamp of the TL review, or `null` if pending.</ResponseField>
    <ResponseField name="reviewed_by" type="integer | null">User ID of the TL who reviewed, or `null` if pending.</ResponseField>
    <ResponseField name="exercise_title" type="string">Title of the exercise.</ResponseField>
    <ResponseField name="language" type="string">Programming language of the exercise.</ResponseField>
    <ResponseField name="difficulty" type="string">Exercise difficulty level.</ResponseField>
    <ResponseField name="topic" type="string">Topic the exercise covers.</ResponseField>
    <ResponseField name="solution" type="string">The reference solution for the exercise.</ResponseField>
    <ResponseField name="expected_output" type="string">Expected output description.</ResponseField>
    <ResponseField name="day_number" type="integer">Plan day the exercise belongs to.</ResponseField>
    <ResponseField name="coder_name" type="string">Full name of the coder who submitted.</ResponseField>
    <ResponseField name="coder_id" type="integer">User ID of the submitting coder.</ResponseField>
    <ResponseField name="plan_id" type="integer">ID of the learning plan the exercise belongs to.</ResponseField>
  </Expandable>
</ResponseField>

### Error responses

| Status | Condition |
| ------ | --------- |
| `400`  | The authenticated TL has no clan assigned. |
| `401`  | No active session. |
| `403`  | User is not a TL. |
| `500`  | Failed to fetch submissions. |

## Review a submission

```
POST /api/tl/submissions/:id/review
```

Adds TL feedback to a submission and awards the coder Kairo points. A real-time notification is sent to the coder.

<Warning>
  You can only review submissions from coders in your own clan. Attempting to review a submission from another clan returns `404`.
</Warning>

### Path parameters

<ParamField path="id" type="integer" required>
  The submission ID to review.
</ParamField>

### Request body

<ParamField body="feedbackText" type="string" required>
  TL's feedback on the submission. Cannot be empty or whitespace-only.
</ParamField>

### Response — 200 OK

<ResponseField name="success" type="boolean">`true` on success.</ResponseField>

<ResponseField name="submission" type="object">
  The updated submission record.
  <Expandable title="properties">
    <ResponseField name="id" type="integer">Submission ID.</ResponseField>
    <ResponseField name="tl_feedback_text" type="string">The stored feedback text.</ResponseField>
    <ResponseField name="reviewed_at" type="string">ISO 8601 timestamp of the review.</ResponseField>
  </Expandable>
</ResponseField>

### Error responses

| Status | Condition |
| ------ | --------- |
| `400`  | `feedbackText` is missing or empty. |
| `401`  | No active session. |
| `403`  | User is not a TL. |
| `404`  | Submission not found or belongs to a coder in another clan. |
| `500`  | Failed to review the submission. |

### Example

<CodeGroup>

```bash cURL
curl --request POST \
  --url http://localhost:3000/api/tl/submissions/88/review \
  --header 'Content-Type: application/json' \
  --header 'Cookie: riwi.sid=<session-cookie>' \
  --data '{
    "feedbackText": "Good solution! Consider using aliases to make the query more readable."
  }'
```

</CodeGroup>

```json 200
{
  "success": true,
  "submission": {
    "id": 88,
    "tl_feedback_text": "Good solution! Consider using aliases to make the query more readable.",
    "reviewed_at": "2026-03-14T16:00:00.000Z"
  }
}
```
