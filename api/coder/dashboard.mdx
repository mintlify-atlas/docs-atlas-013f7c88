---
title: "Coder dashboard"
description: "Retrieve the authenticated coder's dashboard data including active plan, progress metrics, and notifications."
---

All coder routes require an active session cookie, the `coder` role, and completed onboarding. Requests that fail any of these checks return `401` or `403`.

<Info>
  The dashboard endpoint makes seven parallel database queries and returns a single consolidated payload. Use it as the primary data source for the coder home screen.
</Info>

---

## Get dashboard data

```
GET /api/coder/dashboard
```

Returns the full dashboard payload: user profile, soft skills scores, Moodle progress, active plan summary, TL feedback notifications, active risk flags, and recent performance tests.

### Response — 200 OK

<ResponseField name="user" type="object">
  <Expandable title="properties">
    <ResponseField name="fullName" type="string">Coder's display name.</ResponseField>
    <ResponseField name="email" type="string">Registered email address.</ResponseField>
    <ResponseField name="clanId" type="string">Assigned clan identifier.</ResponseField>
    <ResponseField name="role" type="string">Always `coder`.</ResponseField>
    <ResponseField name="kairoScore" type="integer">Current Kairo score. Defaults to `50` when no score events exist.</ResponseField>
    <ResponseField name="moduleId" type="integer">Current module database ID.</ResponseField>
    <ResponseField name="moduleName" type="string">Human-readable module name.</ResponseField>
    <ResponseField name="moduleTotalWeeks" type="integer">Total weeks in the current module.</ResponseField>
    <ResponseField name="isModuleCritical" type="boolean">Whether the current module is marked as critical.</ResponseField>
    <ResponseField name="learningStyleCache" type="string">Cached learning style value (`visual`, `auditory`, `kinesthetic`, `mixed`).</ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="softSkills" type="object | null">
  `null` if the coder has not yet completed the soft skills assessment.
  <Expandable title="properties">
    <ResponseField name="autonomy" type="integer">Score 1–5.</ResponseField>
    <ResponseField name="timeManagement" type="integer">Score 1–5.</ResponseField>
    <ResponseField name="problemSolving" type="integer">Score 1–5.</ResponseField>
    <ResponseField name="communication" type="integer">Score 1–5.</ResponseField>
    <ResponseField name="teamwork" type="integer">Score 1–5.</ResponseField>
    <ResponseField name="learningStyle" type="string">Enum: `visual`, `auditory`, `kinesthetic`, `mixed`.</ResponseField>
    <ResponseField name="assessedAt" type="string">ISO 8601 timestamp of the assessment.</ResponseField>
    <ResponseField name="average" type="number">Mean of the five skill scores, rounded to 2 decimal places. `null` if no scores exist.</ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="progress" type="object">
  Sourced from the `moodle_progress` table. Defaults to week 1 for new coders with no Moodle data.
  <Expandable title="properties">
    <ResponseField name="currentWeek" type="integer">Current week number. Defaults to `1`.</ResponseField>
    <ResponseField name="averageScore" type="number | null">Moodle average score. `null` (not `0`) when no data exists, so the UI can distinguish between no data and a real zero.</ResponseField>
    <ResponseField name="strugglingTopics" type="string[]">Array of topic names the coder is struggling with.</ResponseField>
    <ResponseField name="weeksCompleted" type="array">JSONB array of completed week entries.</ResponseField>
    <ResponseField name="weeksCompletedCount" type="integer">Number of completed weeks.</ResponseField>
    <ResponseField name="updatedAt" type="string | null">ISO 8601 timestamp of the last Moodle sync.</ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="activePlan" type="object | null">
  Summary of the coder's active AI-generated learning plan. `null` if no active plan exists.
  <Expandable title="properties">
    <ResponseField name="id" type="integer">Plan database ID.</ResponseField>
    <ResponseField name="targetedSoftSkill" type="string">The soft skill this plan addresses.</ResponseField>
    <ResponseField name="planType" type="string">Plan type extracted from `moodle_status_snapshot` (e.g. `interpretive`).</ResponseField>
    <ResponseField name="generatedAt" type="string">ISO 8601 timestamp of plan generation.</ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="notifications" type="object">
  <Expandable title="properties">
    <ResponseField name="unread" type="integer">Count of unread feedback messages (last 5).</ResponseField>
    <ResponseField name="items" type="object[]">
      <Expandable title="item properties">
        <ResponseField name="id" type="integer">Feedback record ID.</ResponseField>
        <ResponseField name="text" type="string">Feedback message body.</ResponseField>
        <ResponseField name="type" type="string">Feedback type: `weekly`, `activity`, or `general`.</ResponseField>
        <ResponseField name="tlName" type="string">Display name of the Team Leader who sent the feedback.</ResponseField>
        <ResponseField name="createdAt" type="string">ISO 8601 creation timestamp.</ResponseField>
        <ResponseField name="isRead" type="boolean">`false` triggers the notification bell indicator.</ResponseField>
      </Expandable>
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="riskFlags" type="object[]">
  Up to 3 active (unresolved) risk flags.
  <Expandable title="item properties">
    <ResponseField name="level" type="string">Risk level: `low`, `medium`, or `high`.</ResponseField>
    <ResponseField name="reason" type="string">Human-readable explanation of the detected risk.</ResponseField>
    <ResponseField name="detectedAt" type="string">ISO 8601 timestamp of detection.</ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="performanceTests" type="object[]">
  Up to 3 most recent performance test results.
  <Expandable title="item properties">
    <ResponseField name="score" type="number">Test score.</ResponseField>
    <ResponseField name="status" type="string">Test status.</ResponseField>
    <ResponseField name="moduleName" type="string">Module the test belongs to.</ResponseField>
    <ResponseField name="takenAt" type="string">ISO 8601 timestamp.</ResponseField>
  </Expandable>
</ResponseField>

### Error responses

| Status | Condition |
| ------ | --------- |
| `401`  | No active session. |
| `403`  | User is not a coder, or onboarding is incomplete. |
| `500`  | Internal server error loading dashboard data. |

## Get module milestones

```
GET /api/coder/milestones
```

Returns the coder's progress milestones mapped by module and week, joined with Moodle average scores.

### Response — 200 OK

<ResponseField name="milestones" type="object[]">
  <Expandable title="item properties">
    <ResponseField name="module_name" type="string">Name of the module.</ResponseField>
    <ResponseField name="week_number" type="integer">Week number within the module.</ResponseField>
    <ResponseField name="week_name" type="string">Human-readable week label.</ResponseField>
    <ResponseField name="average_score" type="number">Moodle average score for this week.</ResponseField>
  </Expandable>
</ResponseField>

### Error responses

| Status | Condition |
| ------ | --------- |
| `401`  | No active session. |
| `500`  | Failed to fetch milestone data. |

### Example

<CodeGroup>

```bash cURL
curl --request GET \
  --url http://localhost:3000/api/coder/milestones \
  --header 'Cookie: riwi.sid=<session-cookie>'
```

</CodeGroup>

```json 200
{
  "milestones": [
    {
      "module_name": "Bases de Datos",
      "week_number": 1,
      "week_name": "Introducción a SQL",
      "average_score": 85.0
    },
    {
      "module_name": "Bases de Datos",
      "week_number": 2,
      "week_name": "JOINs y relaciones",
      "average_score": 74.5
    }
  ]
}
```
