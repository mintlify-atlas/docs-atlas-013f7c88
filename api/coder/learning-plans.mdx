---
title: 'Learning plans'
description: 'Request, retrieve, and interact with AI-generated personalized learning plans.'
---

Learning plans are 20-day personalized study plans generated by the Python AI service. Each plan targets a specific soft skill and is based on the coder's Moodle progress and soft skills assessment.

<Note>
  Plan generation is asynchronous. `POST /api/coder/plan/request` returns immediately with `requested: true`. Poll `GET /api/coder/plan` a few seconds later to retrieve the generated plan.
</Note>

---

## Get active plan

```
GET /api/coder/plan
```

Returns the coder's current active learning plan, including completed day tracking and the next pending day.

### Response — 200 OK

<ResponseField name="hasPlan" type="boolean">
  `false` when the coder has no active plan. When `false`, `plan` is `null`.
</ResponseField>

<ResponseField name="plan" type="object | null">
  <Expandable title="properties">
    <ResponseField name="id" type="integer">Plan database ID.</ResponseField>
    <ResponseField name="moduleId" type="integer">Module this plan belongs to.</ResponseField>
    <ResponseField name="planContent" type="object">Full JSONB plan content as generated by the AI service. Structure varies by plan type.</ResponseField>
    <ResponseField name="softSkillsSnapshot" type="object">Snapshot of the coder's soft skills scores at plan generation time.</ResponseField>
    <ResponseField name="moodleStatusSnapshot" type="object">Snapshot of Moodle progress at plan generation time.</ResponseField>
    <ResponseField name="targetedSoftSkill" type="string">The soft skill this plan is designed to improve.</ResponseField>
    <ResponseField name="generatedAt" type="string">ISO 8601 timestamp of plan generation.</ResponseField>
    <ResponseField name="completedDays" type="object">Map of completed day numbers to completion metadata. Keys are string integers (`"1"`, `"2"`, …). Each value has `completedAt` (ISO 8601).</ResponseField>
    <ResponseField name="currentDay" type="integer">First incomplete day between 1 and 20.</ResponseField>
    <ResponseField name="completedCount" type="integer">Number of completed days.</ResponseField>
    <ResponseField name="isComplete" type="boolean">`true` when all 20 days are completed.</ResponseField>
  </Expandable>
</ResponseField>

### Error responses

| Status | Condition |
| ------ | --------- |
| `401`  | No active session. |
| `500`  | Failed to load plan data. |

### Example

<CodeGroup>

```bash cURL
curl --request GET \
  --url http://localhost:3000/api/coder/plan \
  --header 'Cookie: riwi.sid=<session-cookie>'
```

</CodeGroup>

```json 200
{
  "hasPlan": true,
  "plan": {
    "id": 17,
    "moduleId": 4,
    "planContent": { "days": [ { "day": 1, "activities": [] } ] },
    "targetedSoftSkill": "autonomy",
    "generatedAt": "2026-03-10T09:00:00.000Z",
    "completedDays": {
      "1": { "completedAt": "2026-03-11T10:30:00.000Z" },
      "2": { "completedAt": "2026-03-12T11:00:00.000Z" }
    },
    "currentDay": 3,
    "completedCount": 2,
    "isComplete": false
  }
}
```

```json 200 (no plan)
{
  "hasPlan": false,
  "plan": null
}
```

## Request a new plan

```
POST /api/coder/plan/request
```

Triggers AI-powered plan generation. The Node.js server fires a request to the Python AI service (`POST /generate-plan`) and returns immediately — the coder does not need to wait for generation to complete.

<Tip>
  After receiving `requested: true`, poll `GET /api/coder/plan` every 2–3 seconds until `hasPlan` is `true`.
</Tip>

No request body is required. The server uses the coder's `current_module_id`, existing soft skills assessment, and Moodle progress to build the generation payload.

### Response — 200 OK

<ResponseField name="requested" type="boolean">`true` when the generation request was successfully dispatched.</ResponseField>
<ResponseField name="message" type="string">Human-readable status message.</ResponseField>

### Error responses

| Status | Condition |
| ------ | --------- |
| `401`  | No active session. |
| `500`  | Failed to initiate plan request. |

### Example

<CodeGroup>

```bash cURL
curl --request POST \
  --url http://localhost:3000/api/coder/plan/request \
  --header 'Cookie: riwi.sid=<session-cookie>'
```

</CodeGroup>

```json 200
{
  "requested": true,
  "message": "Plan en generación. Reintenta en unos segundos."
}
```

## Get plan details

```
GET /api/coder/plans/:planId
```

Returns a specific plan with its full activity list.

### Path parameters

<ParamField path="planId" type="integer" required>
  The plan's database ID.
</ParamField>

### Response — 200 OK

<ResponseField name="plan" type="object">
  Full `complementary_plans` row including all fields, plus:
  <Expandable title="properties">
    <ResponseField name="id" type="integer">Plan ID.</ResponseField>
    <ResponseField name="coder_id" type="integer">Owner coder ID.</ResponseField>
    <ResponseField name="module_id" type="integer">Associated module ID.</ResponseField>
    <ResponseField name="plan_content" type="object">AI-generated plan JSONB.</ResponseField>
    <ResponseField name="priority_level" type="string">Priority: `low`, `medium`, or `high`.</ResponseField>
    <ResponseField name="is_active" type="boolean">Whether this is the currently active plan.</ResponseField>
    <ResponseField name="generated_at" type="string">ISO 8601 generation timestamp.</ResponseField>
    <ResponseField name="activities" type="object[]">
      Activities belonging to this plan, ordered by `day_number`.
      <Expandable title="activity properties">
        <ResponseField name="id" type="integer">Activity ID.</ResponseField>
        <ResponseField name="day_number" type="integer">Day this activity belongs to (1–20).</ResponseField>
        <ResponseField name="title" type="string">Activity title.</ResponseField>
        <ResponseField name="description" type="string">Activity description.</ResponseField>
        <ResponseField name="estimated_time_minutes" type="integer">Suggested time in minutes.</ResponseField>
        <ResponseField name="activity_type" type="string">Type: `guided`, `semi_guided`, or `autonomous`.</ResponseField>
        <ResponseField name="skill_focus" type="string">Soft skill this activity targets.</ResponseField>
      </Expandable>
    </ResponseField>
  </Expandable>
</ResponseField>

### Error responses

| Status | Condition |
| ------ | --------- |
| `401`  | No active session. |
| `404`  | Plan not found or does not belong to the authenticated coder. |
| `500`  | Failed to retrieve plan details. |

### Example

<CodeGroup>

```bash cURL
curl --request GET \
  --url http://localhost:3000/api/coder/plans/17 \
  --header 'Cookie: riwi.sid=<session-cookie>'
```

</CodeGroup>

```json 200
{
  "plan": {
    "id": 17,
    "coder_id": 42,
    "module_id": 4,
    "plan_content": {},
    "priority_level": "high",
    "is_active": true,
    "generated_at": "2026-03-10T09:00:00.000Z",
    "activities": [
      {
        "id": 101,
        "day_number": 1,
        "title": "Repaso de consultas SQL básicas",
        "description": "Practica SELECT, WHERE y ORDER BY.",
        "estimated_time_minutes": 30,
        "activity_type": "guided",
        "skill_focus": "autonomy"
      }
    ]
  }
}
```

## Mark a day as complete

```
POST /api/coder/plan/:planId/day/:day/complete
```

Marks a specific plan day as complete. Updates the `completed_days` JSONB field on the plan and upserts `activity_progress` records for all activities on that day.

Completing a day awards Kairo points. Completing all 20 days awards a bonus.

### Path parameters

<ParamField path="planId" type="integer" required>
  The plan's database ID.
</ParamField>

<ParamField path="day" type="integer" required>
  Day number to mark complete. Must be between `1` and `20`.
</ParamField>

### Response — 200 OK

<ResponseField name="success" type="boolean">`true` on success.</ResponseField>
<ResponseField name="completedDays" type="object">Updated map of all completed days with their `completedAt` timestamps.</ResponseField>
<ResponseField name="completedCount" type="integer">Total number of completed days after this update.</ResponseField>
<ResponseField name="nextDay" type="integer | null">Next incomplete day number, or `null` if the plan is finished.</ResponseField>
<ResponseField name="isComplete" type="boolean">`true` when `completedCount` reaches 20.</ResponseField>
<ResponseField name="kairoScore" type="integer">The coder's updated Kairo score after point awards.</ResponseField>

### Error responses

| Status | Condition |
| ------ | --------- |
| `400`  | `day` is outside the range 1–20. |
| `401`  | No active session. |
| `404`  | Plan not found, inactive, or does not belong to the coder. |
| `500`  | Failed to complete day. |

### Example

<CodeGroup>

```bash cURL
curl --request POST \
  --url http://localhost:3000/api/coder/plan/17/day/3/complete \
  --header 'Cookie: riwi.sid=<session-cookie>'
```

</CodeGroup>

```json 200
{
  "success": true,
  "completedDays": {
    "1": { "completedAt": "2026-03-11T10:30:00.000Z" },
    "2": { "completedAt": "2026-03-12T11:00:00.000Z" },
    "3": { "completedAt": "2026-03-13T09:45:00.000Z" }
  },
  "completedCount": 3,
  "nextDay": 4,
  "isComplete": false,
  "kairoScore": 75
}
```

```json 400
{
  "error": "Día inválido. Debe ser entre 1 y 20."
}
```
